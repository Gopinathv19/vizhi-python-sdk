# Vizhi Python SDK

The official dependency-free Python client for sending chat requests through
the Vizhi AI gateway and collecting model usage metadata.

## Requirements

- Python 3.10 or newer
- A valid Vizhi agent API token, such as `vz_live_...`

## Installation

Install the latest SDK directly from GitHub:

```bash
python3 -m pip install git+https://github.com/Gopinathv19/vizhi-python-sdk.git
```

For production projects, install a specific release tag so SDK updates do not
unexpectedly affect the application:

```bash
python3 -m pip install git+https://github.com/Gopinathv19/vizhi-python-sdk.git@v0.1.0
```

 

Verify the installation:

```bash
python3 -c "import vizhi_sdk; print(vizhi_sdk.__version__)"
```

## Quick Start

```python
import vizhi_sdk as vs

provider = vs.provide_model(
    "vz_live_your_api_token",
    base_url="https://api.your-vizhi-domain.com",
)

answer = provider.chat("Explain Vizhi in one sentence.")

print(answer.content)
print(answer.output_tokens)
print(answer.input_tokens)
print(answer.latency_ms)
```

The SDK sends the token to the Vizhi backend using:

```text
Authorization: Bearer vz_live_your_api_token
```

The Vizhi backend validates the token, reads the model bound to that token, and
routes the request to the correct model provider.

## Secure Configuration

Do not hard-code production tokens in source code or commit them to GitHub.
Store the token and backend URL as environment variables:

```bash
export VIZHI_API_TOKEN="vz_live_your_api_token"
export VIZHI_BASE_URL="https://api.your-vizhi-domain.com"
```

Then read the token in the application:

```python
import os
import vizhi_sdk as vs

provider = vs.provide_model(
    os.environ["VIZHI_API_TOKEN"],
)

answer = provider.chat("What is an AI gateway?")
print(answer.content)
```

 

## Sending Messages

### Plain prompt

```python
answer = provider.chat("Write a short product description.")
```

### Conversation messages

```python
answer = provider.chat(
    [
        {"role": "system", "content": "Answer concisely."},
        {"role": "user", "content": "What is an API gateway?"},
    ]
)
```

### Generation settings

```python
answer = provider.chat(
    "Give me three product names.",
    temperature=0.7,
    max_tokens=200,
)
```

`temperature` must be between `0.0` and `2.0`. `max_tokens` limits the maximum
number of generated tokens.

## Selecting Models

Select the model in Vizhi when generating the model token. SDK callers should
usually pass only the token:

```python
model = vs.provide_model(token)
```

The backend rejects requests if a caller tries to use a model that does not
match the model bound to the token. The older `provide_model(model, token)` form
is still available for agent-token compatibility.

## Answer And Metadata

Every successful `chat()` call returns a `ChatAnswer`:

```python
answer = provider.chat("Hello")

print(answer.content)        # Generated text
print(answer.input_tokens)   # Tokens sent to the model
print(answer.output_tokens)
# Tokens generated by the model
print(answer.total_tokens)   # Input and output token total
print(answer.latency_ms)     # Provider latency in milliseconds
print(answer.model)
print(answer.provider)
print(answer.query_id)
print(answer.agent_id)
print(answer.response_id)
print(answer.finish_reason)
```

Compatibility aliases are also available:

```python
print(answer.answer)         # Same as answer.content
print(answer.token)          # Same as answer.output_tokens
print(answer.token_queried)  # Same as answer.input_tokens
print(answer.time_taken)     # Same as answer.latency_ms
```

Use `answer.raw` when the complete backend response is required.

## Error Handling

```python
import vizhi_sdk as vs

try:
    answer = provider.chat("Hello")
except vs.AuthenticationError as exc:
    print(f"The Vizhi API token was rejected: {exc}")
except vs.APIError as exc:
    print(f"Vizhi request failed: {exc}")
    print(f"HTTP status: {exc.status_code}")
except vs.InvalidResponseError as exc:
    print(f"Unexpected backend response: {exc}")
```

Common causes:

| Error | Likely cause |
|---|---|
| `AuthenticationError` | Missing, invalid, inactive, or revoked model/agent token |
| `APIError: Could not connect` | Incorrect backend URL or unavailable backend |
| `APIError` with HTTP 400 | Unknown model or invalid request settings |
| `APIError` with HTTP 502 | Upstream model provider failed |
| `InvalidResponseError` | Backend response does not match the SDK contract |

## Timeouts

The default request timeout is 60 seconds. It can be changed when creating the
provider:

```python
provider = vs.provide_model(
    token,
    timeout=120,
)
```

## Updating Or Removing The SDK

Update to the latest GitHub version:

```bash
python3 -m pip install --upgrade git+https://github.com/Gopinathv19/vizhi-python-sdk.git
```

Remove the SDK:

```bash
python3 -m pip uninstall vizhi-sdk
```

## Local Development

Clone and install the SDK in editable mode:

```bash
git clone https://github.com/Gopinathv19/vizhi-python-sdk.git
cd vizhi-python-sdk
python3 -m pip install -e .
python3 -m unittest discover -s tests -v
```

## Client Checklist

Before integrating the SDK, a client needs:

1. The production Vizhi backend URL.
2. An active Vizhi agent API token.
3. An allowed model name.
4. Python 3.10 or newer.

Treat Vizhi API tokens like passwords. Never expose them in frontend,
browser-side, or mobile application code. Use the SDK only from trusted
server-side applications.

```bash
git tag -a v0.1.0 -m "Vizhi Python SDK v0.1.0"
git push origin main
git push origin v0.1.0
```

After the tag is available on GitHub, clients can safely pin that version:

```bash
python3 -m pip install git+https://github.com/Gopinathv19/vizhi-python-sdk.git@v0.1.0
```
